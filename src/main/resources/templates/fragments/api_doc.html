REST API Documentation

General information

To perform a REST API call marked with token, you must provide your API token in a HTTP header named .

To perform a REST API call marked with admin, the provided API token must belong to an administrator.

Accounts
/api/v1/account
GET
admin List all accounts
/api/v1/account/username
GET
token Get the account info of username
PUT
token Update the account info of username
/api/v1/account/username/availability
GET
Return whether username is available for use
/api/v1/account/username/refreshToken
POST
token Generate a new API token for username
/api/v1/account/username/hardware
GET
token Check whether username has access to the hardware infrastructure
POST
admin Grant access to the hardware infrastructure for username
DELETE
admin Revoke access to the hardware infrastructure for username
/api/v1/account/username/admin
GET
token Check whether username has admin rights
POST
admin Grant administrator rights for username
DELETE
admin Remove administrator rights for username
/api/v1/account/pending
GET
admin List all accounts with a pending approval request
Adaptation models
/api/v1/adaptationmodel
GET
token List all adaptation models for the current API user (or all for admin)
POST
token Upload a new adaptation model for the current API user
/api/v1/adaptationmodel/id
GET
token Get the current API user's adaptation model with the given id
PUT
token Update the current API user's adaptation model with the given id
DELETE
token Delete the current API user's adaptation model with the given id
Benchmarking
/api/v1/benchmark
GET
token List all benchmark results for the current API user (or all for admin)
/api/v1/benchmark/id
GET
token Get the current API user's benchmark results with the given id
/api/v1/benchmark/new/modelID
POST
token Submit a new benchmark request for the given modelID
